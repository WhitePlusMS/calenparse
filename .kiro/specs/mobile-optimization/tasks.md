# 移动端样式优化实施任务列表

## 概述
本任务列表基于需求文档和设计文档,通过纯 CSS 媒体查询实现移动端样式优化,不修改 HTML 结构和 JavaScript 逻辑。

## 实施原则
- 仅通过 CSS 媒体查询实现
- 不修改组件的 HTML 结构
- 不添加新的 JavaScript 逻辑
- 桌面端样式保持不变
- 渐进增强:平板(≤768px) → 手机(≤480px) → 小屏手机(≤375px)

## 注意事项

1. 文件组织: 所有移动端样式集中在 src/utils/mobile.css 中,便于维护和管理
2. CSS 优先级: 使用 @media 查询时,确保移动端样式优先级高于桌面端
3. 测试驱动: 每完成一个组件的适配,立即在真实设备上测试
4. 渐进增强: 先实现平板适配,再实现手机适配,最后处理小屏手机
5. 性能监控: 持续监控 mobile.css 文件大小,确保不超过 20KB
6. 浏览器前缀: Vite 会自动添加必要的浏览器前缀,无需手动添加
7. iOS 特殊处理: 
   - 输入框字体最小 16px 防止自动缩放
   - 使用 -webkit-overflow-scrolling: touch 优化滚动
   - 移除点击高亮 -webkit-tap-highlight-color: transparent

## 完成标准

所有任务完成后,应满足以下标准:

- 所有移动端设备上触摸目标至少 44-48px
- 文字清晰可读,无需缩放
- 侧边栏在移动端固定在底部
- 浮动输入框不被底部导航遮挡
- 对话框和弹窗充分利用屏幕空间
- 桌面端样式完全不受影响
- CSS 增量不超过 20KB
- 无额外 JavaScript 逻辑
- iOS Safari 14+ 和 Android Chrome 90+ 兼容
